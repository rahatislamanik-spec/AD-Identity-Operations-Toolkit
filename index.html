01 — Overview

Scope and intent

A production-style PowerShell framework that audits common Active Directory identity governance risks in a simulated financial-services environment, mapped to OSFI E-21, CIS Controls v8, and NIST SP 800-53 themes.

Built for a simulated NorthBridge Financial Group environment modelling enterprise AD governance concerns: privilege separation, fine-grained password policies, stale objects, service account sprawl, hybrid identity readiness, and standing privileged access. The repository stays focused on reusable script logic, report templates, and operational runbook design.

Most scripts are read-only audit scripts. Phase 6 includes optional disable, move, and delete workflow switches for lab validation — those actions should only be used in a controlled AD lab or approved change window.

03 — Phase ledger

Ten-phase audit framework

Each phase is an independent PowerShell audit targeting a specific identity risk surface, with detections and report output mapped to governance control areas.

01

Stale account detection

Get-StaleAccounts.ps1

Read-only audit

Identifies user and computer accounts exceeding inactivity thresholds aligned with financial institution standards — a 30-day threshold versus the 90-day enterprise default. Flags accounts by OU, department, and last-logon delta, and outputs an interactive HTML report with a sortable risk table.

Key detections Inactive >30 days Never logged in Disabled accounts retaining group membership Stale pwdLastSet on computer accounts
02

Privileged account audit

Get-PrivilegedAccountAudit.ps1

Read-only audit

Enumerates all Tier 0 and Tier 1 privileged principals across Domain Admins, Schema Admins, Enterprise Admins, Backup Operators, and Account Operators, with recursive nested-group explosion to surface shadow privilege paths.

Key detections Nested-group privilege escalation Admin accounts with no MFA indicator Non-expiring privileged passwords Service accounts in Tier 0 groups
03

Password policy compliance

Get-PasswordPolicyCompliance.ps1

Read-only audit

Audits both the default domain password policy and Fine-Grained Password Policies across all PSOs, mapping policy coverage gaps and identifying accounts that fall outside compliant policy scope.

Key detections PasswordNeverExpires PasswordNotRequired PSO coverage gaps Policy strength vs OSFI E-21 minimums
04

Group membership audit

Get-GroupMembershipAudit.ps1

Read-only audit

Analyses AD security group health across the environment, detecting circular nesting, orphaned groups, over-privileged distribution lists, and groups exceeding membership thresholds.

Key detections Circular group nesting Empty security groups Groups >500 members, no owner Distribution lists nesting security groups
05

Service account governance

Get-ServiceAccountGovernance.ps1

Read-only audit

Enumerates service accounts by convention and SPN registration, flagging Kerberoastable accounts, SPNs registered on user objects, and service accounts operating outside dedicated OUs.

Key detections Kerberoastable accounts Service accounts in privileged groups Non-gMSA without managed password SPN conflicts
06

Inactive object cleanup workflow

Invoke-InactiveObjectCleanup.ps1

Write actions — gated

Implements a staged disable → move → delete remediation model for lab validation. Default behaviour is discovery and preview; write actions require explicit execution switches and should follow review, approval, and backup validation.

Pipeline Discovery Risk classification Dry-run preview Staged disable OU quarantine 30-day hold Deletion with audit log
07

Executive summary report

New-ExecutiveSummaryReport.ps1

Aggregation

Aggregates findings from all phases into a single executive-ready HTML report with risk scoring, trend indicators, and OSFI E-21 control mapping — designed for CISO and audit committee consumption.

Output Risk-scored HTML dashboard Control gap heatmap Remediation priority matrix Print / PDF-friendly layout
08

Hybrid identity & Entra ID sync audit

Get-HybridIdentityAudit.ps1

Read-only audit

Audits the hybrid identity boundary where on-premises AD synchronises to Microsoft Entra ID via Azure AD Connect, detecting privileged accounts synced to the cloud, AADC service account misconfigurations, UPN conflicts, and stale synced accounts active in both environments.

Key detections Privileged accounts synced to Entra ID Azure AD Connect service account audit UPN suffix mismatches adminCount=1 on synced accounts Stale synced accounts ProxyAddress conflicts
09

PIM governance audit

Get-PIMGovernanceAudit.ps1

Read-only audit

Audits permanent privileged role assignments across the domain and identifies Privileged Identity Management governance gaps — accounts that should be converted to just-in-time eligible roles, SDProp adminCount artifacts, non-dedicated admin accounts with permanent privilege, and stale standing privileged access.

Key detections Permanent Domain Admin assignments SDProp adminCount=1 artifacts Non-dedicated accounts, standing privilege Stale permanent privileged accounts Cloud-synced privileged principals Service accounts with permanent admin rights

05 — Quick start

Run the first phase

Requires PowerShell 5.1+ (7.x recommended) with the ActiveDirectory RSAT module on Windows Server 2019/2022 or Windows 10/11. Domain read access is sufficient for audit phases; Phase 6 write actions require delegated or elevated rights.

# Clone the repository
git clone https://github.com/rahatislamanik-spec/AD-Identity-Operations-Toolkit.git
cd AD-Identity-Operations-Toolkit

# Import the AD module (if not auto-loaded)
Import-Module ActiveDirectory

# Run Phase 1 in dry-run mode
.\Phase1-StaleAccountDetection\Get-StaleAccounts.ps1 -DaysInactive 30 -WhatIf

# Run Phase 1 and generate the HTML report
.\Phase1-StaleAccountDetection\Get-StaleAccounts.ps1 -DaysInactive 30 -GenerateReport -OutputPath ".\Reports\"

08 — Limitations & roadmap

Known limits and future validation

Limitations

  • Contains script logic and documentation, not retained screenshots or generated report artifacts beyond Phase 10.
  • Compliance mappings are educational control references, not a formal audit opinion or certification.
  • Hybrid identity and PIM phases use on-premises AD indicators; full Entra ID cloud-side validation requires Microsoft Graph modules and permissions.
  • Phase 6 can disable, move, or delete AD objects when execution switches are supplied — test in a lab first and use formal change control in any real environment.
  • Some detections rely on naming conventions, AD attributes, or available module data and should be tuned per organization.

Future validation plan
